# 관리자 가이드

이 문서는 Mash-Up Android팀 관리자들을 위한 운영 가이드입니다.

<br>

## 초기 세팅

새 저장소에 이 프로젝트를 세팅할 때 아래 작업이 필요합니다.

### 1. GitHub Labels 생성

Issues 탭 → Labels → **New label**에서 아래 두 라벨을 생성합니다.

| 라벨 이름 | 용도 |
|----------|------|
| `create-seminar` | 세미나 자료 등록 이슈에 자동 부착 (워크플로우 트리거) |
| `delete-seminar` | 팀원 또는 관리자가 수동으로 부착하여 자료 삭제 트리거 |

### 2. Actions 권한 설정

Settings → Actions → General → **Workflow permissions**에서
**Read and write permissions**를 선택합니다.

<br>

## 새 기수 추가

새 기수가 시작될 때 이슈 템플릿의 드롭다운 항목을 추가합니다.

`.github/ISSUE_TEMPLATE/seminar.yml` 파일의 `options`에 새 기수를 추가합니다.

```yaml
options:
  - "16기"
  - "17기"  # 추가
```

> 별도로 폴더나 README 파일을 만들 필요가 없습니다.
> 해당 기수의 첫 이슈가 등록될 때 자동으로 생성됩니다.

<br>

## 잘못 등록된 자료 삭제

자료 삭제는 팀원 본인도 직접 진행할 수 있습니다. 팀원이 직접 처리하지 못하는 경우 관리자가 동일한 절차로 처리합니다.

삭제는 **이슈 단위**로 처리되므로, 한 팀원이 여러 세미나를 등록한 경우 삭제하려는 세미나에 해당하는 이슈를 찾아 진행합니다.

**1.** 삭제할 세미나의 이슈(이미 Close된 이슈)로 이동합니다.

**2.** 우측 **Labels** 항목에서 `delete-seminar` 라벨을 추가합니다.

**3.** 자동으로 아래 과정이 진행됩니다.

```
delete-seminar 라벨 부착
   ↓
해당 기수 README에서 자료 행 제거
   ↓
삭제 완료 댓글 작성
```

> **주의:** 삭제는 이름과 주제가 정확히 일치하는 행을 찾아 제거합니다.
> 동일인이 같은 주제로 중복 등록한 경우 첫 번째 항목이 삭제됩니다.

<br>

## 기수 아카이브 전체 삭제

특정 기수의 자료를 모두 삭제하면 해당 기수 폴더와 메인 README의 목차 링크가 자동으로 제거됩니다.

<br>

## 워크플로우 구조

| 파일 | 트리거 | 역할 |
|------|--------|------|
| `archive-seminar.yml` | 이슈 생성 (`create-seminar` 라벨) | 자료 등록 → README 업데이트 → 이슈 Close |
| `delete-seminar.yml` | 라벨 부착 (`delete-seminar` 라벨) | README에서 해당 행 제거 |
